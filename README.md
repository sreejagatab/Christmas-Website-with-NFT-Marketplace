## Overview

Christmas NFT Marketplace is a modern, responsive web application that combines the festive spirit of Christmas with blockchain technology. Users can connect their MetaMask wallets, browse NFTs, participate in auctions, and make direct purchases in a beautifully designed Christmas-themed interface.

### Key Highlights

- **Web3 Integration**: Seamless MetaMask wallet connection
- **Dual Marketplace**: Both direct sales and auction-based NFT trading
- **Responsive Design**: Optimized for desktop, tablet, and mobile devices
- **Real-time Updates**: Live countdown timer and dynamic content
- **Social Features**: Follow users, track activity, and discover trending NFTs

---

## Features

### Frontend Features

| Feature | Description |
|---------|-------------|
| **Wallet Connection** | Connect/disconnect MetaMask with automatic account detection |
| **NFT Marketplace** | Browse, search, buy, and bid on NFTs |
| **Auction System** | Real-time bidding with highest bid tracking |
| **Christmas Countdown** | Live countdown timer to December 25th |
| **Interactive Gifts** | Animated gift boxes with reveal functionality |
| **Responsive Navigation** | Mobile-friendly hamburger menu |
| **Snowfall Animation** | Festive animated snowfall effect |

### Backend Features

| Feature | Description |
|---------|-------------|
| **User Management** | Registration, profiles, social connections |
| **NFT Operations** | Minting, ownership tracking, metadata management |
| **Marketplace Logic** | Direct sales and auction management |
| **Bid System** | Place, track, and manage auction bids |
| **Search Engine** | Full-text search across NFTs, users, auctions |
| **Activity History** | Comprehensive activity tracking and feeds |
| **Transaction Records** | Blockchain transaction logging |

### Marketplace Statistics

The platform tracks and displays:
- Total registered users
- Total NFTs minted
- Active listings (direct sales)
- Active auctions

### Schema Details

#### User Schema
```javascript
{
  name: String,                    // Display name
  bio: String,                     // User biography
  email: String,                   // Email address (optional)
  avatar: String,                  // Profile picture URL
  banner: String,                  // Profile banner URL
  website: String,                 // Personal website
  twitter: String,                 // Twitter handle
  instagram: String,               // Instagram handle
  discord: String,                 // Discord username
  isVerified: Boolean,             // Verification status
  isFeatured: Boolean,             // Featured user flag
  followers: [ObjectId],           // Array of follower user IDs
  following: [ObjectId],           // Array of following user IDs
  createdAt: Date,                 // Registration timestamp
  updatedAt: Date                  // Last update timestamp
}
```

#### NFT Schema
```javascript
{
  name: String,                    // NFT name
  description: String,             // NFT description
  image: String,                   // Image URL
  metadata: Object,                // Additional metadata
  tokenId: String,                 // Blockchain token ID
  contractAddress: String,         // Smart contract address
  creator: ObjectId,               // Reference to creator user
  createdAt: Date                  // Minting timestamp
}
// Indexes: Text index on 'name' and 'description' for search
```

#### Auction Schema
```javascript
{
  nft: ObjectId,                   // Reference to NFT
  seller: ObjectId,                // Reference to seller user
  highestBidder: ObjectId,         // Reference to highest bidder
  startPrice: Number,              // Starting bid price
  currentPrice: Number,            // Current highest bid
  minIncrement: Number,            // Minimum bid increment
  startTime: Date,                 // Auction start time
  endTime: Date,                   // Auction end time
  status: String,                  // 'active', 'ended', 'cancelled'
  createdAt: Date                  // Creation timestamp
}
```

#### Buy (Direct Sale) Schema
```javascript
{
  nft: ObjectId,                   // Reference to NFT
  seller: ObjectId,                // Reference to seller user
  buyer: ObjectId,                 // Reference to buyer user (when sold)
  price: Number,                   // Listing price
  status: String,                  // 'active', 'sold', 'cancelled'
  listedAt: Date,                  // Listing timestamp
  soldAt: Date                     // Sale timestamp (when sold)
}
```

#### Bid Schema
```javascript
{
  auction: ObjectId,               // Reference to auction
  bidder: ObjectId,                // Reference to bidder user
  amount: Number,                  // Bid amount
  createdAt: Date                  // Bid timestamp
}
```

#### History Schema
```javascript
{
  user: ObjectId,                  // Reference to user
  activity: String,                // 'list', 'purchase', 'bid', 'follow'
  nft: ObjectId,                   // Reference to NFT (optional)
  auction: ObjectId,               // Reference to auction (optional)
  buy: ObjectId,                   // Reference to buy listing (optional)
  relatedUser: ObjectId,           // Reference to related user (optional)
  createdAt: Date                  // Activity timestamp
}
```

## API Documentation

### Base URL

```
Development: http://localhost:4000/api
Production:  https://your-domain.com/api
```

### Response Format

All API responses follow this format:

```javascript
// Success Response
{
  "success": true,
  "data": { ... },
  "message": "Operation successful"
}

// Error Response
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Error description"
  }
}
```

## Installation

### Prerequisites

- **Node.js** (v18 or higher)
- **npm** (v8 or higher)
- **MongoDB** (v6 or higher)
- **MetaMask** browser extension

### Quick Start

1. **Clone the repository**
   ```bash
   git clone https://github.com/your-username/christmas-nft-marketplace.git
   cd christmas-nft-marketplace
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Set up environment variables**

   Create a `.env` file in the root directory:
   ```env
   # Server Configuration
   PORT=4000
   ENV=Development

   # Database
   MONGODB_URI=mongodb://localhost:27017/nft_marketplace

   # Frontend
   REACT_APP_API_URL=http://localhost:4000/api

   # JWT Secret (for authentication)
   JWT_SECRET=your-super-secret-key-here
   ```

4. **Start MongoDB**
   ```bash
   # If using local MongoDB
   mongod

   # Or use MongoDB Atlas connection string in .env
   ```

5. **Run the application**
   ```bash
   # Run both frontend and backend
   npm start

   # Or run separately
   npm run start-server  # Backend only (port 4000)
   npm run start-client  # Frontend only (port 5173)
   ```

6. **Access the application**
   - Frontend: http://localhost:5173
   - Backend API: http://localhost:4000/api

### Build for Production

```bash
# Create production build
npm run build

# The build output will be in the /build directory
```

## Configuration

### Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `PORT` | Backend server port | `4000` |
| `ENV` | Environment mode | `Development` |
| `MONGODB_URI` | MongoDB connection string | `mongodb://localhost:27017/nft_marketplace` |
| `REACT_APP_API_URL` | API base URL for frontend | `http://localhost:4000/api` |
| `JWT_SECRET` | Secret key for JWT tokens | Required |

### Frontend Configuration

**API Base URL** (`src/services/api.js`):
```javascript
const API_BASE_URL = process.env.REACT_APP_API_URL || 'http://localhost:4000/api';
```

**Christmas Countdown Date** (`src/components/countdown/Countdown.js`):
```javascript
const deadline = "December 25, 2025"; // Update annually
```

### Backend Configuration

**Server Settings** (`server/index.js`):
```javascript
const PORT = process.env.PORT || 4000;
const MONGODB_URI = process.env.MONGODB_URI || 'mongodb://localhost:27017/nft_marketplace';
```

**Body Parser Limits** (`server/app.js`):
```javascript
app.use(bodyParser.json({ limit: '50mb' }));
app.use(bodyParser.urlencoded({ limit: '50mb', extended: true }));
```

**Session Configuration** (`server/app.js`):
```javascript
app.use(cookieSession({
  maxAge: 24 * 60 * 60 * 1000, // 24 hours
  keys: [process.env.JWT_SECRET]
}));
```

## Usage

### Connecting Wallet

1. Click the "Connect Wallet" button in the navbar
2. MetaMask will prompt for connection approval
3. Approve the connection request
4. Your wallet address will display in the navbar

### Browsing NFTs

1. Navigate to the Marketplace section
2. Use tabs to switch between "Buy Now" and "Auctions"
3. Use the search bar to find specific NFTs
4. Click on an NFT card to view details

### Buying an NFT

1. Ensure your wallet is connected
2. Browse to the "Buy Now" tab
3. Find an NFT you want to purchase
4. Click the "Buy" button
5. Confirm the transaction in MetaMask
6. Wait for transaction confirmation

### Placing a Bid

1. Ensure your wallet is connected
2. Browse to the "Auctions" tab
3. Find an auction you want to participate in
4. Enter your bid amount (must be higher than current + minimum increment)
5. Click "Place Bid"
6. Confirm the transaction in MetaMask

### Interacting with Gifts

1. Scroll to the Gifts section
2. Click on any gift box
3. Watch the reveal animation
4. Discover the hidden item inside

3. **Input Sanitization**: Add comprehensive input sanitization
4. **HTTPS**: Use HTTPS in production
5. **Environment Variables**: Never commit `.env` files
6. **JWT Rotation**: Implement token refresh mechanisms

## Troubleshooting

### Common Issues

**MetaMask Not Detected**
```
Error: MetaMask not installed
Solution: Install MetaMask browser extension from https://metamask.io
```

**MongoDB Connection Failed**
```
Error: MongoNetworkError
Solution: Ensure MongoDB is running locally or check your connection string
```

**Port Already in Use**
```
Error: EADDRINUSE
Solution: Kill the process using the port or change the port in .env
```

**CORS Errors**
```
Error: Access-Control-Allow-Origin
Solution: Check CORS configuration in server/app.js
```
